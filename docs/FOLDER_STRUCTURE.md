# Правила ведения папок с лабораторными

## Структура твоей папки

Каждый студент должен создать папку в директории `works/` с названием, совпадающим с твоим логином или ФИО.

```
works/
└── <ТВ-ЛОГИН>/
    ├── lab1/
    ├── lab2/
    ├── lab3/
    ├── lab4/
    └── lab5/
```

## Структура каждой лабораторной

Каждая папка лабораторной должна содержать:

```
lab<N>/
├── README.md          # Описание решения
├── Makefile           # или CMakeLists.txt
├── src/               # Исходные файлы .cu, .cpp, .h
│   ├── main.cu
│   ├── kernel.cu
│   └── ...
├── include/           # Заголовочные файлы (опционально)
│   └── ...
├── data/              # Тестовые данные (если нужны)
│   └── ...
└── build/             # Не коммитить! (смотри .gitignore)
```

## Требования к файлам

### README.md для каждой лабораторной

Каждый файл `README.md` должен содержать:

```markdown
# Lab N: <Название лабораторной>

## Описание
Краткое описание задачи и что было сделано.

## Компиляция
```bash
# Команды для компиляции
make
# или
mkdir build && cd build && cmake .. && make
```

## Запуск
```bash
# Команды для запуска
./program [аргументы]
# Примеры вывода
```

## Результаты
- Описание результатов тестирования
- Производительность (если релевантно)
- Примеры работы программы

## Замечания
- Любые особенности реализации
- Известные ограничения
```

### Makefile или CMakeLists.txt

**Makefile пример:**
```makefile
NVCC = nvcc
CFLAGS = -O2 -arch=sm_70

TARGET = program

all: $(TARGET)

$(TARGET): src/main.cu src/kernel.cu
	$(NVCC) $(CFLAGS) -o $(TARGET) $^

clean:
	rm -f $(TARGET) *.o

.PHONY: all clean
```

**CMakeLists.txt пример:**
```cmake
cmake_minimum_required(VERSION 3.20)
project(lab1 LANGUAGES CUDA CXX)

set(CMAKE_CUDA_STANDARD 17)
set(CMAKE_CUDA_ARCHITECTURES 70 80 90)

add_executable(program src/main.cu src/kernel.cu)
```

## Правила .gitignore

Никогда не коммитьте:
- ✗ `.o`, `.cu.o` - объектные файлы
- ✗ `.out`, `program`, `*.exe` - исполняемые файлы
- ✗ `build/` - папка сборки
- ✗ `*.a`, `*.so` - библиотеки
- ✗ Большие файлы с данными (> 10 МБ)
- ✗ Файлы с логами и результатами профилирования

## Коммит-сообщения

Используй понятные, описательные сообщения коммитов:

✅ **Хорошо:**
```markdown
> Добавлена Lab 1: CUDA Graphics — отрисовка треугольника
> Исправлена синхронизация ядра в Lab 2
> Оптимизированы передачи памяти в операторе Собеля
```

❌ **Плохо:**
```
> fix bugs
> add code
> update files
> test
> ahah lol
```

## Процесс разработки

1. Создай локальную ветку для работы
2. Разрабатывай в своей папке в `works/<ТВ-ЛОГИН>/`
3. Пушьте в свой форк
4. Когда работа готова → создавай Pull Request

## Примеры хорошей структуры

### Lab 1: CUDA Graphics

works/john_doe/lab1/

├── Makefile
├── src/
│   ├── main.cpp      # Главная программа

```
